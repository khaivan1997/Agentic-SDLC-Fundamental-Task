================================================================================
  STEP 1 – START THE ENVIRONMENT
================================================================================

1.1  Open a terminal.

1.2  Navigate to the AgentGarage directory:
       cd /Users/khai.a.ngo/Projects/Training/agent-garage

1.3  Start Docker containers.
     You are on macOS (Apple Silicon), so you CANNOT expose GPU to Docker.
     You have two options:

     OPTION A – Run Ollama inside Docker on CPU (slower, simpler):
       docker compose --profile cpu up

     OPTION B – Run Ollama natively on your Mac (faster, recommended):
       a) Install Ollama from https://ollama.com/ if not already installed.
       b) Start Ollama natively:  ollama serve
       c) Pull the model:         ollama pull llama3.2
       d) Start AgentGarage WITHOUT the Ollama container:
            docker compose up
       e) After the containers are running, you must update the Ollama
          connection inside N8N so it points to your Mac host:
            - Go to http://localhost:5678/home/credentials
            - Click on "Local Ollama service"
            - Change the Base URL to: http://host.docker.internal:11434/
            - Save

1.4  Wait for all containers to start. You will see:
       "Editor is now accessible via: http://localhost:5678/"

     The following services will be available:
       - N8N           → http://localhost:5678
       - OpenWebUI     → http://localhost:3000

================================================================================
  STEP 5 – BUILD YOUR OWN WORKFLOW (The Actual Assignment)
================================================================================

The assignment requires you to build a SIMPLE N8N workflow that:
  ✅ Includes at least one API request to the local LLM
  ✅ Has a traceable input/output flow (Webhook, file, etc.)
  ✅ Has prompt design (how N8N "talks" to the LLM)

----- RECOMMENDED USE CASE: PR Description Generator -----

This is the simplest to implement and ties directly to your Task Manager project.

5.1  In N8N, click "+ Add Workflow" (top-right) to create a new workflow.

5.2  Add Node 1 – Webhook (Trigger):
     - Search for "Webhook" in the node panel and add it.
     - Set HTTP Method to POST.
     - Set Path to something like: pr-description
     - This gives you a URL: http://localhost:5678/webhook/pr-description

5.3  Add Node 2 – Ollama Chat Model (LLM Call):
     - Search for "AI Agent" or "Ollama" in the node panel.

     OPTION A (Simple – HTTP Request node):
       - Add an "HTTP Request" node.
       - Method: POST
       - URL: http://ollama:11434/api/generate
         (If running Ollama natively: http://host.docker.internal:11434/api/generate)
       - Body (JSON):
         {
           "model": "llama3.2",
           "prompt": "You are a senior software engineer. Generate a clean, professional Pull Request description based on these commit messages:\n\n{{ $json.body.commits }}\n\nWrite a title, summary, and list of changes.",
           "stream": false
         }
       - Connect the Webhook node output to this node's input.

     OPTION B (Using built-in AI Agent node):
       - Add an "AI Agent" node.
       - Connect it to an "Ollama Chat Model" sub-node.
       - Set the model to "llama3.2:latest".
       - In the System Message, write:
           "You are a senior software engineer. When given commit messages,
            generate a clean, professional Pull Request description with a
            title, summary, and list of key changes."
       - Map the user message to: {{ $json.body.commits }}

5.4  Add Node 3 – Respond to Webhook (Output):
     - Search for "Respond to Webhook" and add it.
     - Map the response body to the LLM's output.
     - This sends the generated PR description back as the HTTP response.

5.5  Connect the nodes: Webhook → HTTP Request/AI Agent → Respond to Webhook

5.6  Save and Activate the workflow.

================================================================================
  TROUBLESHOOTING
================================================================================

Problem: N8N shows "could not connect to Ollama"
  → If running Ollama natively on Mac, make sure the credential Base URL
    is set to http://host.docker.internal:11434/ (not localhost).
  → If running Ollama in Docker (--profile cpu), make sure the URL is
    http://ollama:11434/ (the Docker container hostname).

Problem: LLM response is very slow
  → llama3.2 on CPU can be slow. Be patient (30-60 seconds).
  → Consider using a smaller model like "phi3" for faster responses.

Problem: OpenWebUI shows blank page or timeout
  → Reload the page. OpenWebUI sometimes has display delays.
  → Check that the openwebui container is running: docker ps

Problem: Containers won't start
  → Make sure Docker Desktop is running on your Mac.
  → Check for port conflicts: lsof -i :5678 and lsof -i :3000
  → Try: docker compose down && docker compose --profile cpu up
